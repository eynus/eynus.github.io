js-深拷贝与浅拷贝

在回答这个问题前，我们先回顾一下JS中两大数据类型

  • 基本类型 Undefined、Null、Boolean、Number、String
  • 引用类型 Object Array

    基本类型

    基本类型就是值类型，存放在栈(stack)内存中的简单数据段，数据大小确定，内存空间大小可以分配

    引用类型

    引用类型，存放在堆(heap)内存中的对象，变量实际保存的是一个指针，这个指针指向另一个位置(变量在堆内存中的真正地址)
下面我们对这两个类型分别做一个copy处理
我们会发现 ——>
  1. obj3拷贝了obj.name这个基本类型之后，与obj.name是完全独立的。obj3修改之后，obj.name没有变化。
  2. obj2拷贝了obj这个引用类型之后，当obj2被修改，obj的值也发生了变化，实际上他们操作的是同一个对象。

问题在于：
引用类型被拷贝之后，修改拷贝后的值竟然会影响原来的值。
这是因为对一个对象类型使用‘=’号赋值之后，拷贝的实际上只是一个指针(地址)
正是因为引用类型存在这样的问题，所以有了浅拷贝和深拷贝的概念。

浅拷贝

概念：
对于字符串类型，浅拷贝是对值的复制
对于对象来说，浅拷贝是对对象地址的复制，也就是拷贝的结果是两个对象指向同一个地址

对象中使用一层for循环一一赋值到新的对象中，属于一层的拷贝，也是浅拷贝。所以当对象或数组中存在嵌套结构时，浅拷贝仍然存在有指针拷贝的问题。

深拷贝

概念：
深拷贝开辟一个新的栈，两个对象对应两个不同的地址，修改一个对象的属性，不会改变另一个对象的属性

如何实现深拷贝

1.JSON.parse和JSON.stringify
JOSN对象中的stringify可以把一个js对象序列化为一个JSON字符串，parse可以把JSON字符串反序列化为一个js对象，通过这两个方法，也可以实现对象的深复制

但是JSON方法也有缺点
缺点①
通俗点说，JSON.parse(JSON.stringfy(X))，其中X只能是Number, String, Boolean, Array, 扁平对象，即那些能够被 JSON 直接表示的数据结构。

  1. 无法正确处理Function，undefined -> 丢失
  2. 数据NaN、Infinity ->变为null
  3. 无法处理Date类型 -> 变为字符串
  4. 无法处理RegExp、Error对象 -> 变为空对象
  5. 构造函数生成的对象 -> 丢失其constructor
