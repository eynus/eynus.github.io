内置类型

  1. JavaScript的基本类型有几种？

    javascript的基本类型分为七种内置类型，七种内置类型又分为两大类型：基本类型和对象object（包括null和function）

    基本类型：

    • null
    • undefined
    • boolean
    • string
    • number
    • symbol
  2. (var定义所有变量动态声明解释型语言——js的值有类型，变量没有类型)
  3. typeof()
    1. typeof (foo)——“undefined” 宽容的语言：虽然未被定义，但仍打印undefined
    2. typeof(123)——“number”
    3. typeof(“abc”)——“string”
    4. typeof(null)——“object”,一个bug
    5. typeof typeof 2——“string”
  4. 特殊的值

    1. NaN ——不合法的数 类型转换时出现（背景：历史遗留问题，当时没有报错try catch处理）

    2. NaN + - * / 任何数都等于NaN

    3. NaN与自己不相等

    4. Number()隐式转换

    5. +0 -0

    1. var只声明一次

内置方法

历史背景：1995年出现javascript在前端验证数据，减轻服务器负担。管理层喜欢java,所以招人编写javascript的时候创建这些内置方法。

string、boolean、number不建议使用new的方法创建，返回的是对象，直接使用字面量的形式

function:直接执行和new一个function是一样的

array: 不推荐new的方式，直接使用字面量的形式(更快)

object: 不推荐new的方式，直接使用字面量的形式(更快)

正则表达式：new 不 new 都一样

原型链prototypes

“函数和对象，对象和对象的关系”

对象是被构造函数创建出来的，构造函数用new调用

一个对象和它的原型相连接，而不是基于原型构建的对象

原型：是两个对象之间关系的指向，在创建对象的时候就有原型属性

原型的用处：当前对象找不到属性，可以去原型上找

怎么获取原型的方法：

  1. _ _ proto _ _
  2. a1.constructor.prototype //a1是对象 foo.prototype //foo是个function
  3. getPrototypeOf()
