实践笔记-musicPlayer&打地鼠-this,prototype

重构音乐播放器以及参考面对对象版打地鼠总结：

1.this

绑定事件/计时器的函数里面，this指向会变化。这时需要在外面var me = this保存一份this.然后在事件里面使用me来替代this.

2.prototype

Function.prototype的函数里面想要用Function的变量x,就需要在Function把变量改为x.this，并在prototype里使用this.x

prototyp里面可以使用全局中的变量（就是index.html即引入该js到的文件的中的变量）

3.音乐播放器相关(直播后总结)

  1. 切歌时，src也可以是相对地址
  2. 加载好资源之后，先load，再play()
  3. 切歌时，获取资源需要加载完成才能获取到，那么资源变化放在哪里监听？
    1. window.onload 加载dom文档，但就触发一次，切歌不会出发onload
    2. oncanplay 可以，但是在拖拽进度条也会触发
    3. onloadedmetadata 资源的原始数据加载完成之后触发
    4. 我用的是ondurationchange
  4. 切歌时，先要清空计时器
  5. 优化，创建bindEvent方法可以专门存储事件，初始化时执行一次。
